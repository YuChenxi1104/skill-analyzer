# structure-slide-html.md

本文档只负责约束“结构图最终应该怎么展示”。

它不负责：
- 决定结构图里画哪些目录和文件
- 改写结构图内容文案
- 让 Agent 每次从零重新设计一套结构页

结构图内容选择规则请看 `visual-breakdown.md`。

## 产物定位

结构图默认是中级模块里的 `2.1 结构图`。

规则：
- 默认嵌入合并页面
- 默认在当前页面里直接阅读和操作
- 不再把“点击按钮后跳新页面”作为主体验
- 不再把弹层、iframe、二次页面作为默认方案

如果为了调试保留单独结构图 HTML，可以保留，但它不是默认对外交付。

## 页面目标

用户打开页面后，应该立刻明白两件事：

1. 这是一张结构图
2. 它可以直接在当前区域里缩放、拖动、阅读

## 默认交互

结构图默认支持：
- 滚轮缩放
- 拖动平移
- 双击回到初始视角

规则：
- 所有操作都在当前页面内完成
- 不要求用户先点“放大查看”
- 不要求用户先进入第二层查看器

## 不要的交互

默认不要这些内容：
- 放大按钮
- 缩小按钮
- 方向按钮
- 新页面跳转
- 弹层查看
- iframe 查看
- 一堆工具栏按钮

如果确实需要说明操作，只保留一句轻提示，例如：
- `拖动平移 · 滚轮缩放 · 双击重置`

这句提示必须很轻，不要喧宾夺主。

## 版式规则

### 结构图区域

结构图区域应该：
- 直接占住足够大的阅读空间
- 优先保证图本身可读
- 尽量减少上下大面积留白

不要这样做：
- 图只有一小块，四周大量空白
- 图做成“先看缩略图，再点开看大图”
- 为了按钮或说明文字压缩主体图

### 默认视角

默认视角的目标不是“把整张图完整缩得很小塞进去”，而是：
- 用户一打开就能开始读
- 根节点和主干要清楚
- 同时能看到主要分支关系

如果整图太小看不清，优先：
1. 扩大结构图区域
2. 调整默认视角
3. 调整画布与间距

不要优先加按钮。

## 渲染要求

结构图尽量用可缩放矢量方式绘制。

规则：
- 优先 SVG
- 放大后文字和连线要保持清晰
- 不要依赖一张模糊截图来承载主要文字

## 文本规则

结构图中的文字必须继续服从这些约束：
- 说明文字自动换行
- 英文文件名、扩展名、路径片段尽量完整
- 说明文字太长时先压缩文案，不要靠布局硬救

## 模板与脚本分工

### 内容层

`visual-breakdown.md` 负责：
- 画什么
- 每个节点写什么
- 文案压缩到什么程度

### 展示层

`assets/templates/structure-slide-template.html` 负责：
- 同页内直接交互的结构图容器
- 缩放、拖动、默认视角
- 提示语位置和视觉轻重

### 生成层

`scripts/render_structure_slide.py` 负责：
- 把结构数据写进模板
- 输出最终 HTML
- 给出重叠、越界等 warning

## 验收清单

交付前至少确认：

- 结构图在当前页面内直接可操作
- 没有新页面、弹层、iframe
- 没有额外按钮堆砌
- 放大后文字和连线清楚
- 主图区留白不过量
- 默认视角能直接开始读
- 提示语足够轻
- 没有 icon 重叠

## 反模式

不要这样做：

- 首页是一张静态缩略图，用户必须再点一次才真正能看
- 结构图区域里堆很多按钮
- 把阅读动作拆成“先点按钮，再点弹层，再调缩放”
- 主体图很小，四周全是空白
- 结构图本身没问题，却因为按钮、说明块、装饰块被压坏

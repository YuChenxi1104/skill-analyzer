# 拆解表字段口径

这份文件只讲每一块内容应该怎么写、怎么对外呈现。

## 页面通用规则

- 页面是给最终用户看的，不是给内部协作看的
- 不出现内部标签、版本备注、过程说明、开发者视角文案
- 不写“用户视角”“我们内部看”“实现方式”“规则来源”这类话
- 文案尽量短、尽量白话、尽量让 AI 小白也能直接看懂

## 基本信息

### Skill 名称

写 skill 正式名称。

### Skill 来源

写清来源生态，优先公开地址。

推荐写法：
- `GitHub · 用户名/仓库名`

### 热度

热度由拆解 Agent 实时查询 GitHub，并记录查询日期。

规则：
- 热度信息用 icon 表达，不要做成后台面板风格
- 不要出现生硬英文标签堆叠
- 推荐展示：
  - 星标
  - Fork
  - 查询日期

### 一句话说清楚

这一块替代原来的“解决问题列表”。

目的：
- 帮用户一眼判断这个 skill 值不值得继续看
- 直接说明核心优势
- 直接点出典型场景

写法要求：
- 只写一句完整的话
- 不要拆成 3 到 5 个理由
- 不要写通用废话
- 不要写技术实现

推荐结构：
1. 先说它主要解决什么
2. 再说它的核心优势
3. 最后补一句典型场景

示例方向：
- `适合做风格统一、审美在线、输出稳定的网页 PPT，尤其适合商务汇报、方案分享、发布展示这类场景。`

### 不适合你 (not_for)

**核心判断标准：这是 skill 天生做不了 / 做得不好的事，不是用户提示词写得不对的问题。**

每一条 `not_for` 都应该让用户读完立刻明白：**"这不是我要找的工具，不用进去了。"**

规则：
- 必须从 **skill 能力边界** 出发，不是从"用户需求太复杂"出发
- 描述 **skill 本身不支持的输出格式、协作方式、功能范围**
- 不要写"想完全自定义XXX"这种用户侧描述——那是提示词层面的事，不是 skill 的边界
- 控制在 **2~3 条**，每条是一句完整的判断句
- 如果用户边界模糊（比如"不确定是否支持嵌入视频"），优先写进 `not_for`，不要让用户踩坑

推荐写法（按 skill 类型）：

| Skill 类型 | not_for 关注点 | 示例 |
|-----------|----------------|------|
| **输出格式型** (PPT/文档) | 不支持的**文件格式**、**协作方式** | `需要用 PowerPoint 原生格式继续协作的人` |
| **工具自动化型** | 不支持的**平台**、**账号体系** | `只使用内网环境、无法访问外部 API 的人` |
| **创作生成型** (图片/视频) | 不支持的**精度要求**、**版权场景** | `需要商用级像素精度、不能接受 AI 生成瑕疵的人` |
| **流程规范型** (creator/PM) | 不支持的**团队规模**、**流程阶段** | `单人独立开发、不需要分层管理流程的人` |

反面示例（❌ 不要这样写）：
- `想完全自定义数据来源的人` — 这是需求侧描述，不是 skill 边界
- `提示词写不清楚的人` — 这是用户能力问题，不是 skill 问题
- `想做成咨询系统` — 跟 skill 本身无关

正面示例（✅ 应该这样写）：
- `需要 .pptx 文件格式、不能用 HTML 交付的人`
- `需要多人实时在线协同编辑同一份文件的人`
- `需要在无网环境下运行的人`

## 体验流程

默认用时间线布局，但只保留两个大阶段：

- `准备阶段`
- `结束阶段`

不再在初级模块里展示执行阶段。

### 准备阶段
#### 示例提示词 (prompt_text)

示例提示词放在准备阶段卡片内、输入卡片下方。

规则：
- 提供一段可直接复制的完整提示词
- 提示词内容可以详细，但标题要短（如"试试这样起手"）
- 右上角放 icon 复制按钮
- 复制按钮嵌在提示词正文所在图层，不单独漂浮在模块外
- 不要写"点击复制"等多余说明小字

##### 占位符铁律

| 约束 | 值 | 说明 |
|------|-----|------|
| **必填占位符上限** | **≤ 2 个** | 超过 2 个 = 上手门槛过高 |
| 可选占位符上限 | 1 个 | 用圆括号 `()` 标注，有默认值 |
| 总占位符数 | **≤ 3 个** | 含可选 |

##### 智能预填策略

**必须从同文件的以下字段推导默认值，禁止凭空编造：**

| prompt_text 需要的信息 | 从哪个字段提取 | 提取方式 |
|----------------------|---------------|---------|
| 这个 skill 做什么 | `one_liner` | 提取核心动词+对象（如"做网页PPT演示"） |
| 典型页数/规模 | `hero_subtitle` + `deliveries` | 推导合理范围 |
| 风格/模式默认值 | `one_liner` 中的形容词 | 如"现代简约""流畅动画" |
| 用户需要提供的 | `required_cards` | 转化为引导语 |

##### 模板公式

```
帮我用 {skill_name} {核心动作}。{默认参数组合}。【{唯一必填项：用户内容/主题}】。
```

##### 正反示例

❌ **Bad — 4个裸占位符，全靠用户猜：**
```
帮我用 frontend-slides 做一个【演讲主题】的演示，大约【5-10】页，
风格偏【专业沉稳 / 活泼大胆 / 简约学术】，内容我准备好了，
是关于【你的内容概要】。
```

✅ **Good — 智能预填 + 仅1处必填：**
```
帮我用 frontend-slides 做一份 8~12 页的网页 PPT 演示。
风格偏现代简约 + 流畅动画效果。【在此贴你的内容或一句话描述主题】。
输出一个可直接双击打开的 HTML 文件。
```

✅ **Good — 2个必填（复杂 skill 无法压缩到1个时）：**
```
请用 pm-skills 帮我推进一个 PM 任务。
我处在【discovery / strategy / execution】阶段，目标是【简要描述目标】。
使用 Claude Code 执行，已有素材直接贴在后面。
```

##### 不同 skill 类型的模板倾向

| Skill 类型 | prompt_text 特征 | 示例关键词 |
|-----------|-----------------|-----------|
| **输出型** (PPT/文档/报告) | 强调输出格式 + 页数 + 风格默认值 | "做一份 X 页的...风格偏..." |
| **工具型** (自动化/搜索) | 强调触发条件 + 输出格式 | "帮我监控...汇总成..." |
| **流程型** (creator/analyzer) | 强调阶段 + 目标 + 约束 | "我想用...推进...处在...阶段" |
| **配置型** (follow/setup) | 强调当前环境 + 偏好设置 | "帮我设置...我用的是..." |

### 结束阶段

结束阶段固定拆成三个子区：

1. `核心交付`
2. `如何浏览`
3. `如何修改与分享`

规则：
- 三块内容要清楚分开
- 不要出现大段小字说明
- 每块都用 1 句到 2 句能读完的话表达

#### 核心交付

只回答：
- 最终拿到什么，按照以下分类，可能有多产物

  - 纯数据与结构化文件（给系统/数据库用）
    - 表格类：.csv, .xlsx（Excel），常用于数据分析报告。
    - 交换格式：.json, .xml, .yaml，常用于API接口对接、配置文件生成。
    - 数据库脚本：.sql（直接生成建表语句或数据插入语句）。

  - 静态文档与图文报告（给人类阅读）
    - 办公文档：.docx（Word）、.pptx（PPT）。
    - 排版与网页：.pdf、.md（Markdown）。
    - 思维导图：.mmap 或直接输出 Markdown格式的层级列表（可复制进Xmind生成导图）。

  - 交互式前端应用
    - 单页应用 (.html)：所有CSS和JS都内嵌在一个文件中。常用于交付数据看板、小工具、游戏等。
    - 可视化图表 (HTML + ECharts/D3.js)，比如AI生成带有动态交互（鼠标悬停显示数值、缩放）的图表，比静态图片更有说服力。
    - 原型稿 (HTML)：直接生成高保真的移动端或PC端UI界面，用于给客户确认布局。
    - SVG图形：本质是HTML中的矢量代码，用于交付流程图、架构图、组织结构图，放大不失真。

  - 可执行的代码工程（给程序员/运维）
    - 项目压缩包：.zip / .tar.gz，包含完整的项目文件夹（如 src/, requirements.txt, Dockerfile）。
    - 特定语言文件：.py（Python脚本）、.js（Node.js接口）、.java、.jar（编译好的包）。
    - 基础设施即代码 (IaC)：Terraform 配置或 Kubernetes 的 .yaml 编排文件，AI直接帮你生成服务器部署脚本。

  - 多媒体与专业领域格式（给设计师/工业）
    - 图片/矢量：.png, .jpg（Midjourney类），.svg（矢量，AI画Logo或图标），.eps（印刷级）。
    - 3D模型：.stl, .obj, .step（AI生成用于3D打印或工业设计的立体模型）。
    - 音频/视频：.mp3（语音合成）、.srt（字幕文件）、.mp4（生成视频脚本+分镜）。


推荐写法：
- `交付一个可直接打开、可直接演示的网页 PPT HTML。`

#### 如何浏览

只回答：
- 怎么打开
- 怎么看

推荐写法：
- `双击打开 HTML，就能在浏览器里直接翻页演示。`

#### 如何修改与分享

只回答：
- 后续怎么继续改
- 怎么继续用

推荐写法：
- `后续可以继续改标题、替换图片、更新数据，再导出或演示。`

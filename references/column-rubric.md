# 拆解表字段口径

这份文件只讲每一块内容应该怎么写、怎么对外呈现。

## 页面通用规则

- 页面是给最终用户看的，不是给内部协作看的
- 不出现内部标签、版本备注、过程说明、开发者视角文案
- 不写“用户视角”“我们内部看”“实现方式”“规则来源”这类话
- 文案尽量短、尽量白话、尽量让 AI 小白也能直接看懂

## 基本信息

### Skill 名称

写 skill 正式名称。

### Skill 来源

写清来源生态，优先公开地址。

推荐写法：
- `GitHub · 用户名/仓库名`

### 热度

热度由拆解 Agent 实时查询 GitHub，并记录查询日期。

规则：
- 热度信息用 icon 表达，不要做成后台面板风格
- 不要出现生硬英文标签堆叠
- 推荐展示：
  - 星标
  - Fork
  - 查询日期

### 一句话说清楚

这一块替代原来的“解决问题列表”。

目的：
- 帮用户一眼判断这个 skill 值不值得继续看
- 直接说明核心优势
- 直接点出典型场景

写法要求：
- 只写一句完整的话
- 不要拆成 3 到 5 个理由
- 不要写通用废话
- 不要写技术实现

推荐结构：
1. 先说它主要解决什么
2. 再说它的核心优势
3. 最后补一句典型场景

示例方向：
- `适合做风格统一、审美在线、输出稳定的网页 PPT，尤其适合商务汇报、方案分享、发布展示这类场景。`

### 不适合你

只回答：
- 什么样的人不适合直接用它
- 什么情况下不要把它当成捷径

规则：
- 直接说风险，不要委婉绕圈
- 优先提示“看起来很强，但你现在可能用不上”的情况
- 如果这是框架型或二次开发型 skill，要明确写出来
- 尽量写成 2 条可直接落地的判断句，不要写空泛提醒

推荐写法：
- `想完全自定义来源`
- `想做成自己的咨询系统`

## 体验流程

默认用时间线布局，但只保留两个大阶段：

- `准备阶段`
- `结束阶段`

不再在初级模块里展示执行阶段。

### 准备阶段
只回答：
- 使用前最低限度需要准备什么

推荐内容：
- 运行环境
- 平台依赖
- 必要账号或权限
- 用户需要提供的输入类型

规则：
- 只写真正影响上手的东西
- 不要把所有可能素材都列一遍
- 如果能开箱即用，就直说“几乎不用准备”

#### 示例提示词

示例提示词放在准备阶段卡片内、输入卡片下方。

规则：
- 提供一段可直接复制的完整提示词
- 提示词内容可以详细，但标题要短
- 右上角放 icon 复制按钮
- 复制按钮嵌在提示词正文所在图层，不单独漂浮在模块外
- 不要写“点击复制”等多余说明小字

占位符要求：
- 用 `【】` 标出需要用户替换的部分
- 占位符命名要像用户真的能看懂的内容，不要写技术词

### 结束阶段

结束阶段固定拆成三个子区：

1. `核心交付`
2. `如何浏览`
3. `如何修改与分享`

规则：
- 三块内容要清楚分开
- 不要出现大段小字说明
- 每块都用 1 句到 2 句能读完的话表达

#### 核心交付

只回答：
- 最终拿到什么，按照以下分类，可能有多产物

  - 纯数据与结构化文件（给系统/数据库用）
    - 表格类：.csv, .xlsx（Excel），常用于数据分析报告。
    - 交换格式：.json, .xml, .yaml，常用于API接口对接、配置文件生成。
    - 数据库脚本：.sql（直接生成建表语句或数据插入语句）。

  - 静态文档与图文报告（给人类阅读）
    - 办公文档：.docx（Word）、.pptx（PPT）。
    - 排版与网页：.pdf、.md（Markdown）。
    - 思维导图：.mmap 或直接输出 Markdown格式的层级列表（可复制进Xmind生成导图）。

  - 交互式前端应用
    - 单页应用 (.html)：所有CSS和JS都内嵌在一个文件中。常用于交付数据看板、小工具、游戏等。
    - 可视化图表 (HTML + ECharts/D3.js)，比如AI生成带有动态交互（鼠标悬停显示数值、缩放）的图表，比静态图片更有说服力。
    - 原型稿 (HTML)：直接生成高保真的移动端或PC端UI界面，用于给客户确认布局。
    - SVG图形：本质是HTML中的矢量代码，用于交付流程图、架构图、组织结构图，放大不失真。

  - 可执行的代码工程（给程序员/运维）
    - 项目压缩包：.zip / .tar.gz，包含完整的项目文件夹（如 src/, requirements.txt, Dockerfile）。
    - 特定语言文件：.py（Python脚本）、.js（Node.js接口）、.java、.jar（编译好的包）。
    - 基础设施即代码 (IaC)：Terraform 配置或 Kubernetes 的 .yaml 编排文件，AI直接帮你生成服务器部署脚本。

  - 多媒体与专业领域格式（给设计师/工业）
    - 图片/矢量：.png, .jpg（Midjourney类），.svg（矢量，AI画Logo或图标），.eps（印刷级）。
    - 3D模型：.stl, .obj, .step（AI生成用于3D打印或工业设计的立体模型）。
    - 音频/视频：.mp3（语音合成）、.srt（字幕文件）、.mp4（生成视频脚本+分镜）。


推荐写法：
- `交付一个可直接打开、可直接演示的网页 PPT HTML。`

#### 如何浏览

只回答：
- 怎么打开
- 怎么看

推荐写法：
- `双击打开 HTML，就能在浏览器里直接翻页演示。`

#### 如何修改与分享

只回答：
- 后续怎么继续改
- 怎么继续用

推荐写法：
- `后续可以继续改标题、替换图片、更新数据，再导出或演示。`
